Dataset Overview

Data pulled from PostgreSQL via SQLAlchemy, joining six tables — users, subscriptions, engagement, support tickets, billing, and churn labels — all linked by user_id. Result: 10,000 rows × 21 columns.

Data Type Corrections

On initial load all five date columns were object (string). Cast before analysis:

date_cols = ['signup_date', 'start_date', 'cancellation_date', 'last_renewal_date', 'next_renewal_date'] df[date_cols] = df[date_cols].apply(pd.to_datetime)

Key Findings & Implications

Summary of the most important EDA findings and their implications for the modelling pipeline.

#FindingImplication
1Strong class imbalance: 76.7% Renews, 3.7% Cancels EarlyUse class weighting; evaluate with F1 / ROC-AUC per class, not accuracy
2days_since_last_watch r = +0.623 — far above all other featuresExpected top feature; validate via SHAP post-training
3Most continuous features have near-zero linear correlation with churnNon-linear XGBoost interactions may capture patterns Pearson misses
4Monthly subscribers churn at a higher rate than yearlybilling_cycle is a meaningful categorical signal
5Price increase in last 6m is strongly associated with churnprice_increase_last_6m is a high-value boolean feature
6No missing values in any predictive featureNo imputation needed; exclude cancellation_date from model inputs
7Date fields required casting from string to datetime64Pre-processing must cast types before any date-based feature engineering
